# Launch checklist

## Before publishing
- confirm README first screen looks good on desktop and mobile
- confirm all export files exist and open cleanly
- confirm `dist/skill.zip` matches `skill/portable-coding-guardrails/`
- attach social preview image in repo settings
- add topics and short description

## First public push
- create `v1.0.0` tag
- publish release notes from `docs/launch/GITHUB_RELEASE_NOTES.md`
- pin the repository if it matters in your profile

## Distribution
- post the LinkedIn blurb
- post the X blurb
- use the Hacker News blurb only if you have a real story to tell in comments

## After launch
- add one real transcript under `docs/demo/`
- add one real screenshot of a repo-level install in use
- replace any placeholder media with real captures
