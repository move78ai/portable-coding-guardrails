# Contributing

## What belongs here

Contributions should improve one of these layers:

- repo-level control files
- project-level exports
- proof material
- launch collateral
- packaging quality
- platform-specific installation guidance

## What does not belong here

Do not add:

- giant prompt dumps
- generic prompting essays
- unsupported capability claims
- benchmark theater
- broad rewrites that weaken the repo-first hierarchy
- fallback-prompt edits that make the fallback look stronger than the primary surfaces

## Change standard

Prefer small, explicit, testable improvements.

Good changes:

- tighten wording without changing intent
- clarify deployment order
- improve platform-specific install fidelity
- add missing proof artifacts or validation steps
- fix packaging defects

Weak changes:

- broad philosophical rewrites
- speculative abstractions
- duplicated docs
- unverified performance claims

## Pull request standard

A pull request should state:

- what changed
- why it changed
- which files were touched
- what was checked
- what remains unverified

## Style

Write plainly.

Keep claims narrow.

Do not present assumptions as evidence.
